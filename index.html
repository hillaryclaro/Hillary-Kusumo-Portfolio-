Rodeo Video Labeling Notes

This guide covers Barrel Racing (bounding boxes only) and Bull/Bareback/Saddle Bronc Riding (bounding boxes and keypoints) in COCO JSON format. Follow these for high-quality, consistent annotations.

Barrel Racing

  • Label at 30fps. Only annotate the actual event between start and finish line.
  • Objects: horse, rider, flag, chalk, barrel (all 3 barrels individually). Rider and horse are separate boxes.
  • Bounding boxes must be tight and axis-aligned. Partial occlusions are allowed; include visible parts.
  • Overlaps: rider overlapping horse is fine. Label each object separately.

Bull / Bareback / Saddle Bronc Riding

  • Label at 30fps. Duration: chute gate opens till 8 seconds or rider falls.
  • Objects: bronc/bull, non-bucking horses, rider, pickup rider. Same ID across frames.
  • Bounding boxes: tight around visible extent. Rider separated from animal if falls.
  • Keypoints: use COCO format. Mark occluded points (visibility=1) if partially hidden.
  • Small objects: apply bounding box and mark “too small” for keypoints.
  • Person keypoints: head, shoulders, elbows, wrists, hips, knees, ankles.
  • Bull/horse keypoints: nose, ears, withers, mid-back, loin, tail base, shoulders, hips, elbows, knees, hooves.
  • Overlapping: assign keypoints to correct object; bounding boxes separate.
  • Motion blur: label normally if visible; if uncertain, mark occluded.
  • Fallen rider: label rider and animal separately with keypoints.

Image & Video Evaluation Project

In this project I applied structured QA workflows to evaluate both images and videos. Focus remained on instruction accuracy, visual quality, and usability.

Image Evaluation

  • Checked defects like banding, distortion, and unrealistic edits
  • Compared outputs against prompts
  • Selected best image based on realism and clarity

Video Evaluation

Evaluation focused on motion quality, audio clarity, and consistency. Video 1 showed smoother transitions across frames. Movement remained stable with no abrupt changes. Audio stayed aligned with visuals and remained clear throughout. Video 2 introduced slight inconsistencies in motion and less stable audio sync. These differences affected realism and overall viewing quality.

Final Decision: Video 1 selected. Reason: smoother motion, stable transitions, and clear synchronized audio.
